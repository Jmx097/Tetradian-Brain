# Analysis: A matter of meta - Episode 43, Tetradian on Architectures

## Executive Summary
No single enterprise architecture framework fits all contexts, as real-world needs in sectors like telecommunications and logistics require custom adaptations. Reference frameworks enable local standards while maintaining compatibility with broader ones, and **metaframeworks**, **metamodels**, and **meta-methodologies** provide recursive guidance for building tailored, compatible frameworks that preserve uniqueness.[6]

## Key Takeaways
- No universal framework exists; all must be adapted to fit specific organizational needs, such as in Australia Post where a custom one was developed.[1]
- **Reference frameworks** act as local adaptations of standards, ensuring compatibility by clearly distinguishing **same** (for learning from others) and **different** (local uniqueness).[1]
- **Meta** concepts are recursive: a **metamodel** models how to build models, a **metaframework** guides framework creation, and a **meta-meta model** creates metamodels.[1]
- Adaptation retains compatibility by starting from existing frameworks, amending based on local context, and using patterns to track changes.[2]
- This approach allows organizations to innovate uniquely while benefiting from shared knowledge across frameworks.[1]

## Actionable Insights / Tutorials
- Start with an existing framework as a base rather than inventing from scratch to retain compatibility.
- Use a **same-and-different** analysis: identify what remains unchanged (for cross-learning) and what must be adapted locally.
